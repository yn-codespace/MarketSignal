# RSS STRUCTURE IMPROVEMENTS

## Goal

Improve the RSS pipeline so headlines stay organized, sources are tracked,
duplicates are avoided, and the output is easy to feed into an LLM.

---

1. Headline Structure

---

Instead of storing titles, links, dates, and descriptions in separate vectors,
bundle them into one structure. This prevents index mismatches and keeps the
data cleaner.

struct Headline {
std::string title;
std::string link;
std::string pubDate;
std::string description;
std::string source;
};

---

2. Main RSS Input Data Structure

---

This stores the URLs being queried, the raw RSS text, parsed headlines,
and metadata like ticker symbol and success status.

#include <vector>
#include <string>
#include <unordered_set>

struct RSSInputData {

```
// ticker associated with the request
std::string ticker;

// list of URLs generated for this ticker
std::vector<std::string> urls;

// raw RSS XML returned from curl
std::string rssData;

// parsed headlines
std::vector<Headline> headlines;

// used to avoid duplicate articles across feeds
std::unordered_set<std::string> seenLinks;

// optional combined text for LLM ingestion
std::string combinedHeadlines;

// success flag
bool success = false;
```

};

---

3. RSS Source Structure

---

Used to generate URLs dynamically.

Example format:
prefix + ticker + postfix

The needsTicker flag allows mixing search feeds
(Google News, Bing) with global feeds (Reuters, CNBC).

struct RssSource {
std::string name;
std::string prefix;
std::string postfix;
bool needsTicker;
};

7. Preparing LLM Input

---

Append headlines into a combined text block while staying under
~3000 characters.

Example format:

Ticker: TSLA

Tesla beats earnings expectations.
Tesla upgraded by Morgan Stanley.
Tesla announces new AI chip initiative.

This combined string can be sent directly to an LLM for sentiment scoring.
